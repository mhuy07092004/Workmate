## Page-by-Page Backend Requirements

### 1. Login Page (`/login`)

**Current Implementation:**
- Validates credentials against hardcoded `SIGN_IN_USERS` object from `user.json`
- Stores `workmate_signed_in`, `workmate_current_user_email`, and `workmate_user_role` in localStorage

**Backend Requirements:**

| Endpoint | Method | Request | Response |
|----------|--------|---------|----------|
| `/api/auth/login` | POST | `{ email, password }` | `{ user: User, token: string }` |
| `/api/auth/register` | POST | `{ email, password, role, fullName, companyName? }` | `{ user: User, token: string }` |

**Fields for Registration:**
- `email` (string, required) - Primary login identifier
- `password` (string, required) - Will be hashed on backend
- `role` (enum: "candidate" | "employer", required)
- `fullName` (string, required) - Candidate's full name or employer's name
- `companyName` (string, optional) - Required only for employer role

## Authentication Flow

### Current (Mock) Flow

```
User enters credentials
        │
        ▼
┌─────────────────────┐
│ login.jsx checks    │
│ SIGN_IN_USERS object│
│ (from user.json)    │
└─────────────────────┘
        │
        ▼
┌─────────────────────────────────────────┐
│ localStorage.setItem('workmate_signed_in', 'true')    │
│ localStorage.setItem('workmate_current_user_email', email)  │
│ localStorage.setItem('workmate_user_role', role)      │
└─────────────────────────────────────────┘
        │
        ▼
  Redirect to /
```

### Target (Real Backend) Flow

```
User enters credentials
        │
        ▼
POST /api/auth/login
{ email, password }
        │
        ▼
┌─────────────────────────────┐
│ Backend validates           │
│ Returns JWT token + user    │
│ { user, token }             │
└─────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────┐
│ localStorage.setItem('workmate_token', token)         │
│ (Remove old keys: workmate_signed_in,   │
│  workmate_current_user_email)          │
└─────────────────────────────────────────┘
        │
        ▼
  Redirect to /
        │
        ▼
Subsequent API calls include:
Authorization: Bearer <token>
```

### Token Validation

```javascript
// Frontend will send token in header:
headers: {
  'Authorization': `Bearer ${localStorage.getItem('workmate_token')}`
}

// Backend should:
// 1. Extract token from Authorization header
// 2. Verify JWT signature
// 3. Decode user ID from payload
// 4. Return 401 if invalid/expired
```

## Code Patterns

### Pattern 1: Converting Mock to Real API

**File:** `src/services/userService.js`

```javascript
// BEFORE (mock)
export function getCurrentUser() {
  const email = getCurrentUserEmail()
  return findUserByEmail(email)
}

// AFTER (real API)
export async function getCurrentUser() {
  const token = getToken()
  const res = await fetch('/api/users/me', {
    headers: { 
      'Authorization': `Bearer ${token}` 
    }
  })
  if (!res.ok) throw new Error('Failed to fetch user')
  return res.json()
}
```

### Pattern 2: API Client Setup

```javascript
// src/services/api.js
const API_BASE_URL = 'http://localhost:5000' // Your backend URL

async function api(endpoint, options = {}) {
  const token = localStorage.getItem('workmate_token')
  
  const config = {
    headers: {
      'Content-Type': 'application/json',
      ...(token && { 'Authorization': `Bearer ${token}` }),
      ...options.headers,
    },
    ...options,
  }
  
  const response = await fetch(`${API_BASE_URL}${endpoint}`, config)
  
  if (response.status === 401) {
    // Token expired or invalid
    localStorage.removeItem('workmate_token')
    window.location.href = '/login'
    return
  }
  
  if (!response.ok) {
    const error = await response.json()
    throw new Error(error.message || 'API request failed')
  }
  
  return response.json()
}

// Usage
export const userApi = {
  getCurrentUser: () => api('/api/users/me'),
  updateProfile: (data) => api('/api/users/me', { 
    method: 'PUT', 
    body: JSON.stringify(data) 
  }),
}
```

### Pattern 3: React Component with API

```javascript
// src/pages/profile.jsx
import { useEffect, useState } from 'react'
import { userApi } from '../services/api.js'

function Profile() {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)
  
  useEffect(() => {
    async function loadUser() {
      try {
        const data = await userApi.getCurrentUser()
        setUser(data)
      } catch (err) {
        console.error('Failed to load user:', err)
      } finally {
        setLoading(false)
      }
    }
    loadUser()
  }, [])
  
  const handleSave = async (formData) => {
    try {
      const updated = await userApi.updateProfile(formData)
      setUser(updated)
      alert('Profile saved successfully!')
    } catch (err) {
      alert('Failed to save profile: ' + err.message)
    }
  }
  
  if (loading) return <div>Loading...</div>
  
  return (
    // ... render form with user data
  )
}
```

## Common Issues & Solutions

### Issue: CORS Errors

**Problem:** Frontend cannot connect to backend due to CORS policy

**Solution:** Enable CORS on your backend:

```python
# Flask example
from flask_cors import CORS

app = Flask(__name__)
CORS(app, origins=['http://localhost:5173'])
```

### Issue: Token Not Persisting

**Problem:** User logged out on page refresh

**Solution:** Ensure token is being stored correctly:

```javascript
// After successful login
localStorage.setItem('workmate_token', response.token)

// Verify storage
console.log(localStorage.getItem('workmate_token'))
```

### Issue: Role-Based UI Not Working

**Problem:** Navigation shows wrong options

**Solution:** Backend must return `role` in user object:

```json
{
  "id": 1,
  "email": "user@user.com",
  "role": "candidate",
  "fullName": "John Candidate",
  ...
}
```
